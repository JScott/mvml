Getting Started

An overview of MVML, it's API's and helpers, and how to integrate it into your website.

Usage

MVML isn't natively supported by browsers so it needs to be parsed into HTML and JavaScript Our API makes that an easy step for you.

Quick

The quickest way to utilize this is to make an API request from an HTML page. We've built a helper script that can handle that for you: 

<script src="http://content.dev.mvml.net:6865/js/mvml-api.js"></script>
<script>
  window.onload = function() {
    MVML.load_string(document.getElementsByTagName("mvml")[0].innerHTML);
  }
</script>
<mvml>
---
title: JavaScript test
player:
  start: (0, 0, 20)
  gravity: 0
scene:
- primitive: sphere
  position: (0, 4, 0)
</mvml>

That's it! Just upload that to your website. You can also load it directly from a hosted MVML file: 

<script src="http://content.dev.mvml.net:6865/js/mvml-api.js"></script>
<script>
  MVML.load_file("http://dev.mvml.net/mvml/spec.mvml");
</script>

Advanced

If you have access to server-side code you can also make a call from there. These examples show how I do it for this very page in NodeJS. It should be pretty similar across frameworks.

Privacy issues? You can also host the MVML parsing server yourself. Check out the Github page for more details.

Syntax

Just like HTML, MVML is built out of a syntax of tags. In order to build worlds, you'll need to understand these.

YAML

Just like HTML is essentially XML, MVML is essentially YAML. This is important because unlike XML, YAML is very strict on indentation. Even missing a space could cause an error which forces us to write legibly. 

# Valid:
title: Hello metaverse!
scene:
- primative: sphere
- primative: box
  color: red

# Broken
 title: Hey metaverse
  scene:
- primative:
sphere
- primative: box
color: red

Components

You create worlds in MVML by stringing together components such as the scene tag or player tag. Knowing how to use these is important.

Unfortunately I don't have full documentation of this yet. Until I do, please read spec.mvml which uses each component and its properties at least once. You can see it rendered here.
